Minimodem is quite a neat program as it makes possible the modulation of data into 
sound using various implementations of FSK. So I decided to use it interactively with simple chat 
script in bash. I also added some options to encrypt data before piping it to minimodem.

This software only handles processing the data (modulation/demodulation) 
the transmission/receiving of it still needs to be done by separate hardware.
A way of transmitting data without having to deal with that much red tape (FCC) would be
through the use of free space optics (lasers) or LEDs. 

There are many different simple designs and the majority of them similar. They 
make use of and audio output transformer and a light sensor such as a solar panel or a 
photo-transitor (better option) then have the input and line out each wired 
to 3.5 mm headphone jacks.

Here is a sample of the config file:

#----------------------------VARIABLE OPTIONS-----------------------------------
# 
# encrypt | set to 'sym' for symmetric encryption with passphrase
#           set to 'key' for asymmetric encryption with public/private keys
#           set to 'off' to disable encryption, messages will be in plain text
#--------------------------------------------------------------------------------
encrypt="off"
#--------------------------------------------------------------------------------
# 
# pass    | This is required if you set the encrypt variable
#          if using sym encryption set a 'passphrase'
#          if using key encryption set a key location example: '/Desktop/public.pem'
#--------------------------------------------------------------------------------
pass="lame-password"
#--------------------------------------------------------------------------------
